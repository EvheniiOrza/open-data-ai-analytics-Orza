## 1. Ресурси, що створюються Terraform
- **Resource Group**: Групує всі ресурси в одну сутність для зручного управління.
- **Virtual Network (VNet) & Subnet**: Створюють ізольовану мережу для VM.
- **Network Security Group (NSG)**: Міжмережевий екран, що дозволяє доступ по портах 22 (SSH), 80 (HTTP) та 8501 (Web UI).
- **Public IP**: Забезпечує доступ до віртуальної машини з інтернету.
- **TLS Private Key**: Автоматично генерує SSH-ключі для безпечного доступу без залежності від локальних файлів.
- **Linux Virtual Machine (DC1ds_v3)**: Обчислювальний ресурс, на якому працює Docker.

## 2. Робота Cloud-init
Сценарій `cloud-init` виконується автоматично при першому запуску:
1. Встановлює системні залежності та **Docker Engine**.
2. Клонує вихідний код проекту з GitHub-репозиторію.
3. Налаштовує змінні оточення у файлі `.env`.
4. Виконує команду `docker compose up -d --build`, яка збирає образи з вашого коду та запускає контейнери.

## 3. Розгортання Docker-проекту
Проект складається з декількох сервісів:
- **db (Postgres)**: Зберігає дані аналітики.
- **data_load / data_quality**: Обробляють CSV-файли.
- **web**: Надає інтерфейс користувачу на базі вашого коду.
Взаємодія налаштована через внутрішню мережу Docker `app_network`.

## 4. Перевірка працездатності
Працездатність системи перевірена двома способами:
1. Перевірка HTTP-статусу через `curl`.
2. Візуальна перевірка веб-інтерфейсу за публічною IP-адресою `20.106.240.95`.
