# Звіт: Контейнеризація модулів проєкту за допомогою Docker

## Мета роботи
Опанувати базові принципи контейнеризації застосунків та організувати запуск модулів пайплайну аналізу даних у локальному Docker-середовищі за допомогою `docker-compose`.

## Архітектура мікросервісів
Монолітний код з Jupyter-ноутбуків було розділено на окремі незалежні сервіси, кожен з яких має власний `Dockerfile`. Керування інфраструктурою здійснюється через `compose.yaml`.

У проєкті реалізовано наступні контейнери:
1. **`analytics_db`** — база даних PostgreSQL для зберігання обробленого набору даних.
2. **`analytics_data_load`** — сервіс, що зчитує вхідний CSV-файл та завантажує його в БД.
3. **`analytics_data_quality`** — сервіс аналізу якості даних (пошук пропусків, дублікатів).
4. **`analytics_data_research`** — сервіс для обчислення базових статистик.
5. **`analytics_visualization`** — сервіс генерації графіків (зберігає результати у `.png`).
6. **`analytics_web`** — веб-інтерфейс (Streamlit) для демонстрації звітів та графіків користувачу.

## Взаємодія компонентів
- **Мережа:** Усі контейнери об'єднані у спільну bridge-мережу `app_network`, що дозволяє Python-скриптам звертатися до бази даних за іменем хоста `db`.
- **Томи (Volumes):** 
  - Директорія `./data` прокинута в контейнер `data_load` для доступу до вхідного файлу.
  - Директорія `./reports` налаштована як спільний том для сервісів `data_quality`, `data_research`, `visualization` та `web`. Це дозволяє аналітичним модулям зберігати згенеровані JSON-звіти та PNG-графіки, а веб-сервісу — миттєво їх зчитувати та відображати.
  - Том `pgdata` використовується для персистентного зберігання даних PostgreSQL.

## Результати виконання (Скріншоти)

### 1. Запуск інфраструктури
Проєкт успішно запускається єдиною командою:
```bash
docker compose up --build
```
На скріншоті `img5.png` видно створене середовище аналітики. Проєкт згруповано, що підтверджує коректне розпізнавання Docker Compose конфігурації.
![img5](reports/figures/img5.png)

### 2. Коректне завершення роботи
```bash
docker compose down
```
Docker успішно зупиняє та видаляє усі 6 контейнерів та мережу, що свідчить про правильне налаштування життєвого циклу без "завислих" процесів.
![img6](reports/figures/img6.png)
